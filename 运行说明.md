# 币安代币筛选器 v2.0 运行说明

## 快速开始

### 1. 首次设置（仅需执行一次）

#### OAuth认证设置
```bash
# 使用conda环境运行OAuth设置
/Users/wenxiangxu/opt/anaconda3/envs/crypto_project/bin/python oauth_setup_v1.0.py
```
这将打开浏览器进行Google账户授权，授权后会自动生成`token.json`文件。

### 2. 手动运行筛选器

```bash
# 使用conda环境运行（推荐）
/Users/wenxiangxu/opt/anaconda3/envs/crypto_project/bin/python binance_token_screener_v2.0.py

# 或使用系统Python
python binance_token_screener_v2.0.py
```

运行时会提示：
- 输入现货代币数量（默认80）
- 输入期货代币数量（默认80）
- 是否离线模式（默认否）
- 是否上传到Google Sheets（默认否）

### 3. 自动定时运行

系统配置为每天早上7:45自动运行。

#### 启动调度器
```bash
# 前台运行（测试用）
./start_simple.sh start

# 后台运行（生产环境）
./start_simple.sh background
```

#### 管理调度器
```bash
# 查看状态
./start_simple.sh status

# 查看日志
./start_simple.sh logs

# 停止调度器
./start_simple.sh stop

# 重启调度器
./start_simple.sh restart
```

## 输出说明

### 文件结构
程序会在项目目录下创建以日期命名的文件夹：
```
币安代币分析结果_YYYYMMDD/
├── Excel文件/
│   └── 币安代币分析_YYYYMMDD_HHMM.xlsx
├── CSV文件/          # 仅在Excel创建失败时使用
├── 分析报告/         # 预留目录
└── 日志文件/         # 预留目录
```

### Excel工作表内容
1. **原始数据** - 所有代币的完整数据
2. **交易量排行** - 按24小时交易量排序
3. **市值排行** - 按市值排序（排除主要币种）
4. **涨跌排行** - 当日涨幅和跌幅前30名
5. **期货专注** - 期货交易详细数据
6. **每日涨幅榜** - 14天历史涨幅榜
7. **低量高市值** - 市值前150中期货交易量最低的35个
8. **低市值高交易量** - 市值/期货交易量比值最低的30个

### Google Sheets
- 每天创建新的Google Sheet
- 名称格式：`Binance_Token_screener_Analysis_YYYYMMDD`
- 包含与Excel相同的所有工作表
- 所有人可查看，仅授权用户可编辑

## 注意事项

1. **网络要求**
   - 需要稳定的网络连接
   - 访问Binance API和Google API
   - 访问CoinGecko API获取市值数据

2. **API限制**
   - 程序已内置延迟以避免超过API限制
   - 如遇到429错误，稍后重试即可

3. **数据更新**
   - 现货和期货数据：实时
   - 市值数据：每小时缓存更新
   - 历史数据：14天滚动窗口

4. **错误处理**
   - OAuth token过期会自动刷新
   - 网络错误会自动重试
   - 所有错误都会记录在日志中

## 常见问题

### Q: 如何修改定时运行时间？
A: 编辑`simple_scheduler.py`第174行的时间设置：
```python
schedule.every().day.at("07:45").do(self.run_screener)
```

### Q: 如何修改分析的代币数量？
A: 运行时会提示输入，或修改`binance_token_screener_v2.0.py`中的默认值：
```python
self.spot_count = 80     # 默认现货数量
self.futures_count = 80  # 默认期货数量
```

### Q: Google Sheets上传失败怎么办？
A: 
1. 检查`token.json`是否存在
2. 重新运行`oauth_setup_v1.0.py`
3. 确保`oauth_credentials.json`有效

### Q: 如何查看历史运行记录？
A: 查看日志文件：
```bash
cat simple_scheduler.log
```

## 技术支持

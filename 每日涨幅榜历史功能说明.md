# 📈 每日涨幅榜历史功能说明 (14天版本)

## 🎯 功能概述

根据您的要求，我已经成功实现了每日涨幅榜的历史记录功能。该功能能够：

1. **自动记录每日涨幅榜**：每天运行时自动保存当日的涨幅榜前20名
2. **保存历史数据**：所有历史数据永久保存在本地JSON文件中
3. **按要求格式显示**：在Google Sheets的"每日涨幅榜"标签页中显示最近14天的历史记录

## 📊 显示格式 (14天版本)

每日涨幅榜显示为29列（排名 + 14组代币/日期）：

| 排名 | 代币 | 日期 | 代币 | 日期 | 代币 | 日期 | ... | 代币 | 日期 |
|------|------|------|------|------|------|------|-----|------|------|
| 1    | NEW1 | 07-22| RDNT | 07-21| ARB  | 07-20| ... | CFX  | 07-08|
| 2    | NEW2 | 07-22| PENDLE| 07-21| OP  | 07-20| ... | DIA  | 07-08|
| 3    | NEW3 | 07-22| LDO  | 07-21| MAGIC| 07-20| ... | OM   | 07-08|
| ...  | ...  | ...  | ...  | ...  | ...  | ...  | ... | ...  | ...  |

- **第1-2列**：最新日期的涨幅榜（代币 + 日期）
- **第3-4列**：前1天的涨幅榜（代币 + 日期）
- **第5-6列**：前2天的涨幅榜（代币 + 日期）
- **...**：依此类推
- **第27-28列**：前13天的涨幅榜（代币 + 日期）

## 🔧 技术实现

### 核心方法

1. **`create_daily_gainers_history(df)`**
   - 创建每日涨幅榜历史数据
   - 自动加载现有历史，添加当日数据
   - 返回格式化的DataFrame用于上传

2. **`load_daily_gainers_history()`**
   - 从本地JSON文件加载历史数据
   - 如果文件不存在则创建新的历史记录

3. **`merge_historical_gainers_data(history_data, current_data, current_date)`**
   - 合并历史数据和当前数据
   - 按照最新14天的数据创建显示格式

4. **`save_daily_gainers_history(merged_data)`**
   - 保存历史数据到本地JSON文件
   - 确保所有历史数据都被保留

5. **`format_gainers_history_for_display(merged_data)`**
   - 格式化数据为29列显示格式（排名 + 14组代币/日期）
   - 确保列顺序符合要求

### 数据存储

- **本地文件**：`daily_gainers_history.json`
- **格式**：JSON格式，按日期组织
- **内容**：每个日期包含当日涨幅榜前20名的完整信息

```json
{
  "2025-07-19": [
    {"排名": 1, "代币": "CFX", "日期": "2025-07-19"},
    {"排名": 2, "代币": "DIA", "日期": "2025-07-19"},
    ...
  ],
  "2025-07-20": [
    {"排名": 1, "代币": "TRUMP", "日期": "2025-07-20"},
    ...
  ]
}
```
